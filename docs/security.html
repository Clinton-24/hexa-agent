Security Overview

How Hexacrimson agents authenticate, encrypt communications, and fit into a zero-trust control plane.

Quickstart API Reference Security CLI Reference

Zero-trust enrollment

Agents enroll with a short-lived token (hxl-…). Tokens are hashed at rest; the control plane never stores raw secrets longer than the request lifecycle.

Signed binaries

Installers verify SHA-256 checksums before activation. Production deployments should pin checksums and serve installers over TLS only.

Transport

  • TLS for all hub traffic (mTLS recommended in production)
  • Heartbeat intervals keep agent status fresh without long-lived reverse shells
  • Module toggles are explicit API calls — no silent privilege escalation

Security Agent add-on

  • Monitoring — CPU, memory, process anomaly signals
  • Threat Detection — signature + behavioral heuristics, quarantine hooks
  • WiFi Intelligence — rogue AP / deauth / SSID spoofing signals on edge segments

Compliance posture

Architecture is designed to support SOC 2, HIPAA, and GDPR control mappings (access logging, encryption in transit, data minimization). This open demo stores agent metadata locally in data/agents.json — replace with a hardened datastore for production.
